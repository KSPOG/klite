KLite client and automation API

A searchable Java reference for reading client state, performing controlled interactions, running automation tasks, and moving through the world with the integrated walker.

This is an in-client Java API, not a public HTTP API.

Use these types from a KLite plugin or automation module running inside the desktop client. The /api/ website path only hosts this documentation.

Start here

Choose the correct API area

Most plugins use the Client API first, add the automation runtime for long-running workflows, and use the web walker only when travel is required.

01

Client API

Reads inventory, players, NPCs, widgets, combat state, camera state, varbits, and other client data. It also exposes controlled interactions such as clicking an NPC, withdrawing an item, or changing a client setting.

Typical entry point: KLiteClientApi
02

Automation runtime

Coordinates repeatable or long-running work. Use it for task lifecycle, cancellation, delays, status reporting, and keeping automation logic separate from UI or plugin startup code.

Use it when an action spans multiple game states or ticks.
03

Web walker

Plans and executes travel to a destination. It handles route progress and movement-specific state so plugin code does not need to repeatedly issue raw tile clicks.

Use WorldPoint destinations with X, Y, and plane.

How to read the reference

Important return types and conventions

CompletableFuture<T>

The call completes asynchronously. Prefer thenApply, thenCompose, or thenAccept instead of blocking the client with get() or join().

Optional<T>

No matching object may exist. Check isPresent(), use ifPresent, or provide a fallback instead of calling get() blindly.

*Snapshot

A point-in-time, read-only view of client state. Request a new snapshot when current information matters; do not treat an old snapshot as a live game object.

KLiteInteractionResult

An interaction request can be accepted, rejected, or fail. Inspect the result before advancing the automation state.

List<T>

A collection of current matches. An empty list is valid and means nothing matched at that moment.

WorldPoint

A world coordinate containing X, Y, and plane. Make sure the plane is correct when targeting objects, ground items, or walker destinations.

Read state without blocking

This example checks whether the inventory contains coins and handles the asynchronous result.

KLiteClientApi api = /* injected API */;

api.inventoryContains(995)
    .thenAccept(hasCoins -> {
        if (hasCoins) {
            // Continue the workflow.
        }
    });
